# Level2 setup

Ниже — как собрать уровень в одной сцене: маленькая комната, потом большая комната, и затем ещё одно действие.

## 1. Общая идея

- Игрок начинает в маленькой комнате.
- В комнате есть дверь-загадка.
- При нажатии на кнопку/триггер открывается окно с примером, например `7 + 9`.
- После правильного ответа:
  - дверь меняет спрайт на открытую,
  - появляется эхо,
  - таймер сбрасывается,
  - игрок возвращается в начало комнаты.
- После этого игрок идёт через открытую дверь в большую комнату.
- В большой комнате уже ставишь второй триггер/кнопку/действие.

## 2. Что создать в сцене

### Игрок
На объекте игрока должны быть:
- `Rigidbody2D`
- `Collider2D`
- `PlayerMovement`
- `PlayerInteraction`
- `PlayerVisionLight`
- Tag = `Player`

### Видимость вокруг игрока
Чтобы видимость была как у космонавта:
1. Убедись, что проект использует URP и есть 2D Renderer.
2. На сцене должен быть `Global Light 2D` с не очень ярким светом.
3. На игрока повесь `PlayerVisionLight`.
4. Настрой у него `Point Light Outer Radius` — это дальность видимости.
5. Чем меньше radius, тем сильнее эффект ограничения видимости.
6. Если хочешь мягкое затемнение по краям, добавь `Vignette` через `Volume`.

Обычно хватает таких значений:
- `Outer Radius = 6..10`
- `Inner Radius = 1`
- `Intensity = 1.2..2`

### TimeManager
На отдельном объекте или префабе поставь один `TimeManager` и не дублируй его в каждой сцене.

В `TimeManager` назначь:
- `Timer Text` — текст таймера
- `Life Hearts` — 3 изображения сердечек
- `Game Over Panel` — меню конца игры
- `Player` — игрок текущей сцены
- `Spawn Point` — точка старта текущей сцены
- `Echo Prefab` — префаб эха

### Как сделать, чтобы сердца сохранялись между уровнями
1. `TimeManager` уже хранит жизни и таймер между сценами.
2. Если на Level1 осталось 2 сердца, при переходе на Level2 тоже будет 2.
3. Для этого в каждой сцене нужно добавить `TimeManagerSceneBinder`.
4. Этот биндер просто подключает UI текущей сцены к уже существующему `TimeManager`.

### Как собрать сердца в Canvas
1. В `Canvas` создай 3 объекта `UI -> Image`.
2. Назови их, например, `Heart1`, `Heart2`, `Heart3`.
3. Поставь каждому спрайт сердца.
4. Размести их рядом с таймером.
5. Добавь пустой объект `SceneBinder`.
6. Повесь на него `TimeManagerSceneBinder`.
7. Перетащи в него:
   - `Timer Text`
   - `Heart1`, `Heart2`, `Heart3`
   - `Game Over Panel`
   - `Player`
   - `Spawn Point`

### Как это использовать
- `TimeManager` должен быть уже создан **один раз** в первой сцене.
- На следующем уровне новый `TimeManager` не создавай.
- Просто поставь новый `SceneBinder` и привяжи к нему местный UI.
- Если на Level1 осталось 2 сердца, то после перехода на Level2 будет тоже 2.
- Если ты нажмёшь Restart, сцена перезагрузится и жизни снова станут 3.
+
+Важно:
+- `SceneBinder` нужен в **каждой** сцене, где есть свой UI.
+- `TimeManager` нужен только **один** на всю игру.
+- У `TimeManager` не должно быть дубликатов в других сценах.

Что происходит:
- если таймер дошёл до нуля, одно сердечко исчезает;
- если сердца ещё остались, петля перезапускается;
- если сердца закончились, появляется меню `Game Over`;
- кнопка `Restart` вызывает `TimeManager.RestartGame()`.

### Меню Game Over
Сделай `Panel` в Canvas и выключи его в начале.
Внутри добавь:
- текст `Game Over`
- кнопку `Restart`

На кнопке `Restart` в `OnClick` повесь:
- объект с `GameOverMenuUI.RestartGame()`

### Дверь в маленькой комнате
На объект двери повесь:
- `SpriteRenderer`
- `Collider2D`
- `Door`
- `MathDoorPuzzle`

В `Door`:
- укажи `Open Sprite` — спрайт открытой двери

### Вход в Level2
Чтобы дверь действительно переносила на Level2:
1. Повесь `DoorLevelTransition` на сам объект двери.
2. Скрипт сам создаст trigger-коллайдер, если его нет.
3. После открытия двери основной коллайдер будет отключён, а trigger останется активным.
4. Когда игрок войдёт в открытую дверь, загрузится сцена `Level2`.

Важно:
- У двери должен быть обычный `Collider2D` для блокировки.
- У этого же объекта должен быть `DoorLevelTransition`.
- Сцена `Level2` должна быть добавлена в Build Settings.

В `MathDoorPuzzle`:
- `Puzzle Panel` — панель UI с примером
- `Question Text` — TMP-текст примера
- `Answer Input` — TMP_InputField
- `Feedback Text` — TMP-текст для ошибок/успеха
- `Use Random Question` — если включено, пример генерируется сам
- `Fixed Left Value` и `Fixed Right Value` — если хочешь задать пример вручную, например `7` и `9`

Как задавать ответ:
- в `Answer Input` игрок просто печатает число с клавиатуры;
- например, если пример `7 + 9`, то в поле нужно ввести `16`;
- кнопку Submit привязываешь к `MathDoorPuzzle.SubmitAnswer()`.

Если хочешь поменять пример вручную:
- выключи `Use Random Question`;
- впиши свои числа в `Fixed Left Value` и `Fixed Right Value`;
- тогда на панели всегда будет именно этот пример.

### UI-панель загадки
Сделай Canvas, внутри:
- panel
- text с примером
- input field
- кнопка Submit
- кнопка Close

На кнопке Submit в `OnClick` вызывай:
- `MathDoorPuzzle.SubmitAnswer()`

На кнопке Close в `OnClick` вызывай:
- `MathDoorPuzzle.ClosePuzzle()`

### Как игрок вводит ответ
Всё делается прямо в `TMP_InputField`:
- игрок кликает по полю;
- пишет число с клавиатуры;
- нажимает кнопку Submit.
- можно также нажимать `Enter` / `Numpad Enter`.

Если не получается печатать в поле:
1. Проверь, что `Puzzle Panel` реально включается (`SetActive(true)`).
2. У `TMP_InputField` должен быть компонент `EventSystem` в сцене.
3. На поле не должен быть выключен `Interactable`.
4. Если UI не получает фокус, кликни мышкой прямо по полю ещё раз.
5. Убедись, что ты используешь именно `TMP_InputField`, а не обычный `InputField`.

Если хочешь, можно потом добавить автопроверку по Enter, но сейчас проще всего через кнопку Submit.

### Как добавить кнопку/триггер
Сейчас удобнее использовать один и тот же `ActionTrigger` и для Level1, и для Level2.

#### Для кнопки Level1:
1. Создай `2D Object -> Sprite` или просто пустой `GameObject`.
2. Назови, например, `ButtonTrigger`.
3. Добавь на него `BoxCollider2D`.
4. Поставь галку `Is Trigger`.
5. Добавь скрипт `ActionTrigger`.
6. В поле `Door To Open` перетащи дверь.
7. Поле `Puzzle To Open` оставь пустым.

#### Для кнопки Level2:
1. Создай отдельный `GameObject`.
2. Добавь на него `BoxCollider2D` с `Is Trigger`.
3. Добавь скрипт `ActionTrigger`.
4. В поле `Puzzle To Open` перетащи объект двери с `MathDoorPuzzle`.
5. Поле `Door To Open` оставь пустым.

Как это работает:
- если у `ActionTrigger` заполнен `Door To Open`, он открывает дверь, спавнит эхо и сбрасывает таймер;
- если у `ActionTrigger` заполнен `Puzzle To Open`, он открывает панель загадки;
- после правильного ответа дверь открывается;
- эхо появляется рядом с кнопкой, а не у двери;
- коллайдер двери отключается.

## 3. Как сделать выход из маленькой комнаты в большую

Самый простой вариант:
- дверь в маленькой комнате стоит между комнатами;
- после решения она становится открытой;
- игрок просто проходит через неё в большую комнату.

То есть сцена одна, но пространство разделено на 2 зоны:
- маленькая комната
- большой зал

## 4. Как добавить второе действие в большой комнате

Во второй комнате поставь любой новый `ActionTrigger`:
- скрытая кнопка
- плитка
- рычаг
- предмет

Если хочешь, чтобы там тоже была дверь:
- добавь второй объект двери
- на него повесь `Door`
- на кнопку укажи эту дверь в `doorToOpen`

## 5. Что происходит при правильном ответе

Текущий код делает так:
- вызывает `Door.Open()`
- вызывает `TimeManager.ActionCompleted(...)`
- спавнит эхо
- сбрасывает таймер
- телепортирует игрока на `spawnPoint`

Поэтому после решения двери игрок появляется в начале, но уже с открытой дверью.

## 6. Что важно проверить

- `spawnPoint` должен стоять в начале маленькой комнаты.
- У двери должен быть `Collider2D`.
- У панели загадки объект `puzzlePanel` должен быть выключен в начале.
- Сцена `Level2` должна быть добавлена в Build Settings.
- Если хочешь именно одну сцену, переход сцены не нужен.
